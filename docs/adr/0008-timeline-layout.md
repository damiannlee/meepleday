# 0008. 첫 화면을 기간 그룹 타임라인으로

- 상태: Accepted
- 맥락: 서비스를 "보드게임 이벤트 캘린더"로 재정의([PRD §1](../prd.md#1-서비스-정체성))하면서 첫 화면의 조직 원리를 다시 정해야 함. 기존 구현은 카드 그리드 + 마감임박순 정렬.

## 결정

- 첫 화면 = **기간으로 그룹핑한 세로 타임라인** (`이번 주` / `이번 달` / `그 이후` / `예고`).
- **날짜가 확정된 이벤트만 본문**에, `ANNOUNCED`(일정 미정 예고)는 **하단 별도 섹션**으로 분리 ([ADR-0004](0004-derived-status.md) 개정).
- 1급 필터 = **이벤트 유형**(펀딩/선주문/특가/행사) + **진행 상태**(예정/진행중/마감임박). 지역·플랫폼은 2급(접힘 영역).
- 지역 필터 기본값은 반드시 **전체** — 국내·해외 통합이 서비스의 명분이므로 기본 분리는 자기부정.
- 정렬은 기본 마감임박순 하나만. 타임라인이 이미 시간순이라 추가 정렬 축은 요청이 관측되면 도입.
- 데스크톱 우선이되 모바일 대응은 필수 — 공유 링크는 실제로 모바일에서 열림.

## 근거

- **"언제"가 서비스의 축이므로 시간이 1급 조직 원리여야 함.** 정체성과 레이아웃을 일치시킴.
- 마감 압박을 가장 직접적으로 전달 → 발견 → 추적 퍼널의 전환을 촉진.
- 날짜 미정 예고를 자연스럽게 수용할 자리가 있음(하단 섹션).
- 유형 필터를 1급으로 둔 이유: 유저 의도가 "살 것 찾기"와 "갈 곳 찾기"로 갈리는데, 이게 카드가 유형별로 변신하는 구조([PRD §4.1](../prd.md#41-이벤트-모델))와도 일치.
- 상태 필터가 기간 그룹과 겹쳐 보이지만 다름 — 그룹은 *언제 끝나나*, 상태는 *지금 참여 가능한가*.

## 기각한 대안

- **카드 그리드 (현행 구현)**: 핫링크 이미지를 가장 잘 살리고 탐색성이 높음. 그러나 시간축이 약해져 "일정 서비스"라는 정체성이 흐려짐. 재정의 이전 기획의 산물이라 교체 대상.
- **월별 캘린더**: "일정을 한 곳에서"에 가장 직관적으로 들어맞아 보이나 세 가지가 깨짐 — ① 펀딩은 기간(시작~마감)이라 점 하나로 안 찍힘, ② 날짜 미정 예고를 놓을 칸이 없음, ③ 스크롤 면적 대비 정보량이 낮아 데스크톱 우선 전제와 상충.
- **타임라인 / 그리드 뷰 토글**: 유저에게 선택권을 주지만 두 레이아웃을 만들고 유지해야 함. 초기 비용 2배에 전환율은 대개 낮음(YAGNI). 요청이 관측되면 재검토.
- **플랫폼을 1급 필터로**: 플랫폼 수가 적어 필터로서 변별력이 낮고 지역 필터와 상당 부분 겹침.

## 트레이드오프 / 후속 과제

- **세로 타임라인은 와이드 화면에서 여백이 뜸.** 데스크톱 우선과 긴장 관계 → 행(row) 기반 고밀도 레이아웃(좌측 썸네일 + 중앙 정보 + 우측 D-day/CTA) + 우측 사이드 영역(필터·예고 요약)으로 해소.
- **따뜻한 톤앤매너**([PRD §5.5](../prd.md#55-톤앤매너))**와 고밀도가 서로 당김** → 색·질감이 따뜻함을, 레이아웃이 밀도를 담당하도록 역할 분리. 여백으로 따뜻함을 내려 하면 밀도가 무너짐.
- 이벤트 유형을 **색만으로 구분 금지** — 아이콘·라벨 병기(접근성).
- 예고 섹션은 `scheduleNote`가 자유 문자열이라 **정렬·필터 불가**. 예고 건수가 늘면 정밀도 필드 도입을 재검토.
- 광고·스폰서 슬롯을 나중에 끼워넣으면 밀도가 무너짐 → 수익화를 고려한다면 레이아웃 설계 단계에서 자리만 잡아둘 것(구현은 별개).
